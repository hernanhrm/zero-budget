# Plan: Fix Sign-In - Set Active Organization on Session

## Problem
When an existing user signs in, the `active_organization_id` is not being set on their session, causing the error: `{"message":"No active organization","code":"NO_ACTIVE_ORGANIZATION"}`

On sign-up, a `databaseHooks.user.create.after` hook creates a personal organization and sets `active_organization_id` on the session. But this hook doesn't run for existing users signing in.

## Solution
Add a `databaseHooks.session.create.after` hook that runs after a session is created (on sign-in) to set the `active_organization_id` if it's not already set.

## File to Modify

### `backend/cmd/identity/src/auth.ts`

Add session.create hook to set active organization:

```typescript
databaseHooks: {
  user: {
    create: {
      after: async (user, context) => {
        // ... existing code for sign-up ...
      },
    },
  },
  session: {  // ADD THIS NEW HOOK
    create: {
      after: async (session) => {
        if (session.activeOrganizationId) return;
        
        // Find the user's first organization
        const userMemberships = await db.select()
          .from(members)
          .where(eq(members.user_id, session.userId))
          .limit(1);
        
        if (userMemberships.length > 0) {
          await db.update(sessions)
            .set({ active_organization_id: userMemberships[0].organization_id })
            .where(eq(sessions.id, session.id));
        }
      },
    },
  },
},
```

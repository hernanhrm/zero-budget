# Deploy Identity Service to Cloudflare Container Apps

## Summary

The identity service is deployed via GitHub Actions to Cloudflare Container Apps (full Node.js container, keeps existing code).

## GitHub Actions Workflow

Created: `.github/workflows/deploy-identity.yml`

Triggers on: Push to `backend/cmd/identity/**` on main branch

## Required GitHub Secrets

| Secret | Description |
|--------|-------------|
| `CF_API_TOKEN` | Cloudflare API token with Container Apps write access |
| `IDENTITY_DATABASE_URL` | Neon PostgreSQL connection string |
| `IDENTITY_BETTER_AUTH_SECRET` | Auth secret (min 32 chars) |
| `IDENTITY_BETTER_AUTH_URL` | Public URL of identity service |
| `IDENTITY_TRUSTED_ORIGINS` | Comma-separated trusted origins |
| `IDENTITY_CORS_ORIGIN` | Allowed CORS origin |
| `IDENTITY_APP_URL` | Frontend URL for email links |
| `IDENTITY_GOOGLE_CLIENT_ID` | Google OAuth client ID (optional) |
| `IDENTITY_GOOGLE_CLIENT_SECRET` | Google OAuth secret (optional) |
| `IDENTITY_GITHUB_CLIENT_ID` | GitHub OAuth client ID (optional) |
| `IDENTITY_GITHUB_CLIENT_SECRET` | GitHub OAuth secret (optional) |

## Required GitHub Variables

| Variable | Description |
|----------|-------------|
| `CF_CONTAINER_REGISTRY_USER` | Your Cloudflare container registry username |

## Setup Steps

1. **Create Cloudflare API Token**
   - Go to Cloudflare Dashboard > My Profile > API Tokens
   - Create a token with `Container Apps: Edit` permission

2. **Add GitHub Secrets**
   - Go to GitHub repo > Settings > Secrets and variables > Actions
   - Add each secret from the table above

3. **Add GitHub Variables**
   - Go to GitHub repo > Settings > Secrets and variables > Actions > Variables
   - Add `CF_CONTAINER_REGISTRY_USER` with your Cloudflare username

4. **First Deployment**
   - Push to main or run workflow manually
   - The container app will be created automatically

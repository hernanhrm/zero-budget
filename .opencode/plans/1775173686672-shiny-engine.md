# Fix PermissionClient Hardcoded Currency Module

## Problem
Line 48-49 in `backend/cmd/api/middleware/permission_client.go` hardcodes `"currency"` as the module:
```go
permReq := permissionsRequest{
    Permissions: map[string][]string{
        "currency": {permission},  // WRONG: permission is "currency:read"
    },
}
```

The `permission` parameter is formatted as `"resource:action"` (e.g., `"currency:read"`), but we need to split it to get the module `"currency"` and action `"read"`.

## Fix

Parse `permission` to extract module and action, then build the request correctly:

```go
// permission is like "currency:read"
parts := strings.Split(permission, ":")
module := parts[0]
action := parts[1]

permReq := permissionsRequest{
    Permissions: map[string][]string{
        module: {action},
    },
}
```

## File: `backend/cmd/api/middleware/permission_client.go`

1. Add `"strings"` to imports
2. Parse permission string to extract module and action

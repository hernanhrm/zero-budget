# Fix: Missing `active_team_id` Column in Sessions Table

## Problem
Error: `column "active_team_id" of relation "sessions" does not exist`

The migration `20260322120001_add_teams.up.sql` (line 21) adds this column but hasn't been run on the database.

## Solution

Run the pending migrations on your database:

```bash
# Run all pending migrations
cd backend/cmd/migrate && migrate -path migrations -database "$DATABASE_URL" up

# Or run just the teams migration
cd backend/cmd/migrate && migrate -path migrations -database "$DATABASE_URL" up 20260322120001
```

## Verification

After running, verify the column exists:

```sql
SELECT column_name FROM information_schema.columns 
WHERE table_schema = 'identity' 
AND table_name = 'sessions';
```

You should see `active_team_id` in the list.

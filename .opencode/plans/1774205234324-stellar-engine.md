# Update Members Page Frontend to Match Pencil Design

## Summary
Update the frontend `members-table.tsx` to match the pencil design, including proper implementation of the role select dropdown for EDITOR/VIEWER roles.

## Current State
- `RoleBadge` uses hardcoded mock data
- For owner: shows yellow dot + "OWNER" text badge (no border) ✓

- Styling doesn't fully match pencil design

## Pencil Design Specs

### Member Row Structure (64px height, padding [0, 24])
- **Avatar**: 36x36, Owner=yellow(#FFD600) bg, Editor/Viewer=gray(#2d2d2d) bg
- **Member info**: Name (13px, #F5F5F0) + Email (10px, #3D3D3D)
- **Role column**: 160px wide
  - OWNER: badge with yellow dot + "OWNER" text (no border)
  - EDITOR/VIEWER: bordered dropdown button (stroke #2d2d2d) with role text + chevron-down
- **Joined**: 140px, text #6B6B6B, font IBM Plex Mono 12px
- **Actions**: 80px, Owner shows "—", Editor/Viewer show trash icon (#FF6B35)

### Role Dropdown Styling
- Height: 24px
- Padding: [0, 12]
- Gap between text and chevron: 6px
- Border: 1px stroke #2d2d2d
- Text: 10px, Space Grotesk, #F5F5F0
- Chevron: 10x10, lucide chevron-down, #6B6B6B

# Simplify Organization Creation Flow

## Summary
Remove the dedicated `/create-organization` page entirely. Instead, automatically create an organization on the backend when a new user is created, using a Better Auth `databaseHooks.user.create.after` hook. The organization will use the user's name as its name, a slug generated from name + random string, and will be set as active automatically via `keepCurrentActiveOrganization: true`.

## Implementation Plan

### 1. Add Database Hook to Auth Configuration

**File:** `backend/cmd/identity/src/auth.ts`

Add a `databaseHooks` configuration to the `betterAuth()` call. Since we're inside the `betterAuth` config, we need to use `context.api` to call the organization creation:

```typescript
databaseHooks: {
  user: {
    create: {
      after: async (user, context) => {
        if (!context) return

        const baseName = user.name ?? user.email ?? 'user'
        const slug = baseName
          .toLowerCase()
          .replace(/\s+/g, '-')
          .replace(/[^a-z0-9-]/g, '') +
          '-' +
          Math.random().toString(36).substring(2, 8)

        // Call the organization create endpoint
        await context.api.call('/organization/create', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            name: baseName,
            slug,
            userId: user.id,
            keepCurrentActiveOrganization: true,
          }),
        } as any)
      }
    }
  }
}
```
